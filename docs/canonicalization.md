# Canonicalization v1

Canonical inputs use UTF-8 without Unicode normalization and SHA-256. Every value is framed by a fixed-width unsigned 16-bit big-endian field ID, a one-byte presence marker, a fixed-width unsigned 32-bit big-endian byte length, and the value. Signed integers use their fixed-width two's-complement big-endian representation. UUIDs use lowercase canonical text. Metadata and required projector IDs are sorted and duplicate metadata keys are rejected. Null and empty are distinct. Payload bytes are length-framed in the request fingerprint; payload digest, length, encoding, and reference are framed in the envelope checksum. Unknown fields are excluded until a new canonicalization version includes them. Deterministic Protobuf serialization is not the contract.

The request fingerprint covers event ID, authenticated producer ID, namespace, logical stream type, event type, schema version, stream key, operation identity, requested minimum durability, payload bytes, canonical client metadata, and canonicalization version. It excludes installation ID, producer instance/sequence, session, timestamps, Agent policy/routing, and allocated journal/stream sequences. The envelope checksum is computed inside the local acceptance transaction after allocation and covers every immutable persisted envelope field. Each mutable snapshot checksum covers state kind, event ID, revision, status, bounded summaries, and relevant evidence. Same revision plus different checksum is an integrity conflict.
