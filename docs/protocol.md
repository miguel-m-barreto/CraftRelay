# Protocol

Sprint 2 extends the v1 Protobuf contracts with producer registration (lifecycle states, priority/quota classes, registered handles, rejection codes), extended IntegrationManifest validation (event/query/namespace declarations, durability/retention/priority/quota classes, schema version ranges, validation codes including BEST_EFFORT rejection and self-promotion rejection), credential/ACL models (credential kinds/statuses, ACL principals/rules/scopes, allow/deny decisions with typed reason codes), policy resolution (admission decisions, rejection reasons including durability weakening/retention weakening/projection bypass/self-promotion/installation escape, effective policy results with policy version and ownership snapshot ID), NODE_LOCAL ownership snapshot (snapshot IDs/versions, namespace entries, ownership validation codes), quota/admission control (per-producer/namespace/global quota states, admission rejection reasons including P0 reserved capacity protection), and profile validation skeletons (Kafka durability profiles preserving RF=5/minISR=5/acks=all for P0, retention/projection/quota profiles, profile validation codes).

Sprint 1 adds bounded handshake, envelope-input, publish-status, typed-query freshness, barrier, authenticated token, and detachable watch request/response shapes. Java validates immutable client intent before transport submission. Installation and producer identity remain authenticated session inputs rather than arbitrary request selectors. The in-memory Agent and Query fixtures are explicitly non-durable and non-authoritative.

The v1 Protobuf skeleton is under `proto/craftrelay/v1`. Positive signed fields use `int32`/`int64`; zero is valid only for Kafka offsets. Repeated collections are policy-bounded. Core queries use typed `oneof` parameters or registered typed schema handles—never SQL or arbitrary schema names. Mutable fields and payload bytes are deliberately absent from `StoredEventEnvelope`.
